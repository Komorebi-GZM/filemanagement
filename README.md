## 本地文件多标记交叉管理系统（v1.0.0）

### 项目简介

本项目是一个基于 PyQt5 的本地文件管理系统，支持文件多标记管理、搜索、预览等功能，采用 MVC 架构设计，使用 SQLite 作为后端数据库。系统在基础管理功能之上，增强了元数据管理能力（文档简介、特定印记）及交互体验（悬浮窗提示）。

### 核心功能

#### 1. 文件管理
- 支持添加本地文件到系统  
- 文件信息自动采集（文件名、大小、路径、哈希值等）  
- 软删除机制，支持文件恢复  
- 右键菜单操作（打开文件、打开位置、添加标记、删除）  

#### 2. 标记管理
- 支持创建、编辑、删除标记  
- 标记支持层级结构（父子关系）  
- 标记树可视化展示  
- 标记右键菜单（重命名、删除）  

#### 3. 文件标记
- 支持为文件添加多个标记  
- 支持从文件中移除标记  
- 基于标记搜索文件  

#### 4. 搜索功能
- 支持按文件名搜索  
- 支持按标记搜索  
- 实时搜索结果展示  

#### 5. 文件预览与元数据
- 支持图片文件预览  
- 支持文本文件预览  
- **文档简介**：支持为文件添加和编辑自定义简介备注  
- **特定印记**：支持记录文件的特定操作印记，并按时间倒序查看历史  

#### 6. 交互优化
- **悬浮窗提示**：鼠标悬停在文件或标记上时，自动显示详细信息提示

### 使用说明

#### 1. 添加文件
- 点击菜单「文件」→「导入…」或按快捷键 `Ctrl+I`  
- 选择本地文件或文件夹  
- 文件会自动添加到系统中  

#### 2. 管理标记
- 在左侧标记树中右键点击标记  
  - 选择「重命名」修改标记名称  
  - 选择「删除标记」删除标记  
- 点击「+」按钮添加新标记  

#### 3. 为文件添加标记

**方法 1：右键菜单**
- 在文件列表中选择文件  
- 右键点击，选择「添加标记…」  
- 选择要添加的标记  

**方法 2：拖拽标记（推荐）**
- 在文件列表中选择一个或多个文件（支持 `Ctrl` 多选、`Shift` 连续多选）  
- 将选中的文件拖拽到左侧标记树的目标标记上  
- 释放鼠标完成标记  

#### 4. 筛选和查看文件
- **全部文件**：默认显示所有文件  
- **按标记筛选**：点击左侧标记树中的标记，只显示该标记下的文件  
- **按类型筛选**：在顶部筛选栏选择「文件类型」下拉框  
- **按位置筛选**：在顶部筛选栏选择「磁盘位置」下拉框  
- **组合筛选**：可以同时使用多种筛选条件  

#### 5. 排序文件
- 在顶部筛选栏选择排序方式：「文件名升序」或「文件名降序」  

#### 6. 搜索文件
- 在搜索框中输入关键词  
- 按回车或点击「搜索」按钮  
- 搜索结果会显示在文件列表中  

#### 7. 重置筛选
- 点击「重置筛选」按钮，清除所有筛选和搜索条件，显示全部文件  

#### 8. 文件预览与元数据编辑
- 选择文件：在文件列表中点击任意文件  
- 查看预览：在右侧预览区域查看文件内容（图片或文本）  
- **编辑简介**：在预览区下方的简介栏中直接输入或修改文件备注  
- **查看印记**：在预览区下方的印记列表中，按时间倒序查看该文件的历史特定印记  

#### 9. 文件操作
- 右键点击文件，选择「打开文件」打开文件  
- 右键点击文件，选择「打开所在位置」打开文件所在文件夹  
- 右键点击文件，选择「删除（软删除）」软删除文件

### 数据库结构

- `file_records`：存储文件记录  
- `tags`：存储标记信息  
- `file_tag_relation`：存储文件与标记的关联关系  
- `file_notes`：存储文件简介备注（新增）  
- `file_imprints`：存储文件特定印记及时间戳（新增）  

---

### 注意事项

- 本系统仅在本地运行，所有数据存储在本地 SQLite 数据库中  
- 支持的预览文件类型：图片（`.png`, `.jpg`, `.jpeg`, `.gif`）、文本文件（`.txt`, `.md`, `.py`, `.json`, `.xml`, `.html`）  
- 软删除的文件可以在数据库中恢复，不会实际删除本地文件  
- 悬浮窗提示：无需额外操作，鼠标悬停即可自动触发
